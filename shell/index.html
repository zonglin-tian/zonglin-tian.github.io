基础知识

  • 脚本前加 set -euo pipefail
  • 空格: 赋值语句 = 两边不能有空格, 测试语句 []内必须有空格分隔
  • 引号
    • 变量, 命令替换, 含空格/特殊字符的字符串 (文件名) 必须加引号
      • 默认用双引号 "" (防止空格/特殊字符问题): 变量扩展, 支持转义字符
      • 不需要变量扩展时单引号 ' (正则表达式): 静态字符, 完全原样输出
    • test 或 [] 条件判断中变量比较必须加引号
  • 命令替换
    • 反引号包围: 简单替换 (反引号与命令字符间没有空格)
    • $(cmd): 复杂替换, 可读性高 (推荐写法)
  • 同时执行多个命令: 将命令放在同一行, 用 ; 隔开 (字符数不超过 255)
  • 输出内容 echo
    • 可用单引号或双引号来划定文本字符串. 如果字符串中有引号, 需要使用另一种引号将字符串划定起来
    • echo -n 输出内容不增加换行符
  • 执行数学运算
    • expr 命令, 特殊操作符需要转义且将结果赋值只一个变量时需要使用命令替换: var2=$(expr $var1 \* 2)
    • $[option] (方括号前后没有空格): var2=$[$var1 * 2]
    • 只支持整数运算, 符点运算可使用 bc 命令, 支持数字, 变量, 注释, 表达式, 编程语句, 函数 (使用 quit 退出)
  • 退出脚本
    • 每个命令都使用退出状态码, 是一个 0~255 的整数
    • $?: 保存上个已执行命令的退出状态码
    • 常见状态码:
      • 0: 命令成功结束
      • 1: 一般性认知错误
      • 2: 不适合的 shell 命令
      • 126: 命令不可执行
      • 127: 没找到命令
      • 130: 通过 Ctrl + C 终止的命令
    • exit 命令: 默认情况下, shell 脚本会以脚本中的最后一个命令的退出状态码退出. exit 命令允许在脚本结束时指定一个退出状态码, 参数中可使用数值与变量

重定向

  • Shell 在运行命令前, 从左到右解析并设置重定向:
    • 处理所有重定向 >, 2>, < (重定向符号指向数据流动的方向)
    • 执行命令 command
  • 标准流:
    • 标准输入 (stdin), 0, <, 默认键盘 (终端输入)
    • 标准输出 (stdout): 1, > 默认屏幕
    • 错误错误 (stderr): 2, 2>, 默认屏幕
    • 标准输出与标准错误: &>, 默认屏幕, 只能接文件名
  • 写入类型: 覆盖写入 (>), 追加写入 (>>)
  • 同时重定向 stdout 与 stderr 到文件:
    • 传统写法: > output.log 2>&1, >> output.log 2>&1
    • 简化写法: &>, &>>
  • 分离重定向 stdout 与 stderr 到文件: > output.log 2> error.log
  • 丢弃输出: 写入 /dev/null
  • 同时输出到终端与文件 (管道 (|) + tee): command 2>&1 | tee output.log
  • 输入重定向: command < input.txt (从文件读取输入)
    • 可以放在命令的任意位置, 运行命令时从其读取输入
    • 统计文件行数: wc -l < input.txt
  • 内联输入重定向 (inline input redirection) <<: 无需使用文件进行重定向, 只需要在命令行中指定用于输入重定向的数据即可. 同时, 需要指定一个文本标记来划分输入数据的开始和结尾. 任何字符串都可作为文本标记, 但在数据的开始和结尾文本标记必须一致, 常用 EOF
  • 分流命令 tee: 将[标准]输出显示到终端并写入文件 (T 型管道)
    • 语法: command [2>&1] | tee 选项 以空格空格的文件列表
    • 选项: -a 追加 (默认覆盖), -i (忽略中断信号), -p (诊断写入错误)
    • 默认只处理 stdout, 不自动捕获 stderr, 可显式使用 2>&1
    • 最常用形式: command 2>&1 | tee output.log
    • 写入两个文件: command | tee file1.log file2.log
    • 写入特权文件: command | sudo tee file.log
    • 不显示输出: command | tee file &> /dev/null

变量

定义

  • 赋值格式: 变量名=值
  • 等号两边不能有空格, 如果值中包含空格或特殊字符, 需用引号包裹 (对于一个简单无空格的单词, 可以不加引号)
  • 变量名只能包含字母, 数字和下划线, 且不能以数字开头
  • 大小写敏感, 通常用全大写加下划线表示表示常量或配置变量 (LOG_DIR), 使用小写+下划线表示普通变量 (user_name)
  • 避免使用 Shell 内置变量或系统环境变量名, 建议给自定义变量加前缀或使用不容易冲突的名称
  • 临时变量可使用前缀 _ 或 temp_以区分普通变量

设置默认值

  • 当 var 未定义或为空时返回 default: ${var:-default} 或 ${var:=default} (同时给 var 赋值)
    • 当 default 不写时, 表示默认为空
  • 当 var1 未定义或为空时返回 var2 的值: ${var1:-$var2}

本地与环境变量

  • 本地变量: 只在当前 Shell (脚本或终端) 有效, 退出后自动销毁
  • 环境变量: 用 export 标记后, 不但在当前 Shell 可见, 还会被 “继承” 到子 Shell 或脚本, 可用 env 或 printenv 查看

声明与删除

declare 和 typeset 是两个功能几乎相同的命令, 常用于变量声明和属性设置: declare 选项 变量名=值

选项 含义 示例
-i 声明为整数 declare -i num=10
-r 声明变量为只读 declare -r name="Tom"
-a 声明为普通数组 declare -a arr=(1 2 3)
-A 声明为关联数组 (哈希表) declare -A map
-p 查看变量类型和值 (调试常用) declare -p name

删除变量: unset var_name

引用

  • 使用 $, 但引用变量进行赋值时不需要美元符
  • 引号:
    • 双引号围绕: 保持原样的引用变量
    • 不加双引号:
      • 变量中有空格会被拆分为多个变量 → for
      • 变量中有通配符, 会先展开 → *
    • 单引号围绕: 纯字符串对待 (内容一点不变)
  • 花括号:
    • 无花括号包围: 简单引用, 后面紧跟空格, 换行或特殊符号
    • 花括号包围:
      • 界定变量名
      • 设定变量为空时的默认值: ${var:-default_value}
      • 获取字串: ${var:ind:num} (从下标 ind 开始, 取 num 个字符)
      • 删除前后缀
      • 计算长度: ${#str}

数组

  • 元素之间通常用空格分隔也可用换行分隔增加可读性
  • 向数组增加元素: array+=(new_ele)

条件表达式

test 命令

  • test 命令允许 if-then 语句中测试不同条件, 而不仅仅是退出状态码
  • 表达式: test condition
    • 如果条件成立, 退出并返回状态码 0
    • 如果条件不成立, 退出并返回状态码 1
  • 可以使用 [空格condition空格] 来代替 test condition 命令
  • test 命令可以判断三类条件: 数值比较, 字符串比较, 文件比较
  • 使用文本代码来表示数值比较
  • 使用标准数学比较符号来表示字符串比较
  • 如果不是很确定一个变量的内容, 最好在将其用于数值或字符串比较之前先通过 -n 或 -z 来测试一下变量是否含有值

数值比较

数值条件测试可以用在数字 (仅整数) 和变量上, 有以下的条件参数:

  • 等于: -eq
  • 大于或等于: ge
  • 大于: gt
  • 小于或等于: le
  • 小于: lt
  • 不等于: -ne

字符串比较

可用的字符串比较功能:

  • 相同: =
  • 不同: !=
  • 小 (需要转义): \<
  • 大 (需要转义): \>
  • 长度非 0: -n str
  • 长度为 0 (未定义变量的长度也为 0): -z str

文件比较

文件比较允许测试 Linux 文件系统上文件和目录的状态, 可用比较符号如下:

  • 存在且是一个目录: -d file
    • 将文件写入目录
    • 准备切换到某个目录
  • 存在: -e file (使用文件或目录前)
  • 存在且是一个文件: -f file
  • 存在且可读: -r file
  • 存在且非空: -s file
  • 存在且可写: -w file
  • 存在且可执行: -x file
  • 存在且属当前用户: -O file
  • 存在且默认组与当前用户相同: -G file
  • 新: file1 -nt file2
  • 旧: file1 -ot file2

组合表达式

使用布尔逻辑来组合测试, 有两种布尔运算符可以使用:

  • AND: [ condition1 ] && [ condition2 ]
  • OR: [ condition1 ] || [ condition2 ]

双括号

双括号命令允许在数值比较过程中使用高级数学表达式, 即使用更多的数学表达式, 格式为: ((空格expression空格)) (可赋值或比较)

双方括号

双方括号命令允许字符串高级比较 (模式匹配), 格式为: [[空格expression空格]]

  • 四种形式:

多分支 (case)

适用一个变量可能有多个值的情况 (elif 不断检查同一个变量)

  • 基本语法
  • case 会按顺序检查每个模式, 第一个匹配的模式会被执行
  • 通过 | 在一行中分隔出多个模式, * 会捕获所有与已知模式不匹配的值
  • 每个分支需要以 ;; 结束 (特殊情况除外)

循环

for

  • 终端使用: for Iteration; do Commands; done
  • 脚本:
    • 遍历列表:
    • C 风格: for (( i=1; i <= 10; i++ ))
      • 变量赋值可以有空格
      • 条件中的变量不以美元符开头
      • 迭代过程的版式未用 expr 命令格式
      • 使用多个变量: for (( a=1, b=10; a <= 10; a++, b-- ))
  • for 与 do 同一行形式: for 变量 in 列表; do
  • 在某个值两边使用双引号时, shell 并不会将双引号当成值的一部分
  • 列表中的值如果有单引号, 可使用转义字符转义或使用双引号圈起来
  • 用空格来划分列表中的每个值, 如果单独的数据值中有空格, 必须用双引号将这些值圈起来
  • 字段分隔符
    • 环境变量 IFS (内部字段分隔符) 默认有 空格, 制表符, 换行符. 如果 shell 在数据中看到了这些字符中的任意一个, 就会假定列表中一个新数据段的开始
    • 可以根据数据修改 IFS 的值, 使其识别特定字符 (多个分隔符直接串起来)
  • 常见循环列表形式:
    • 变量定义: list="a b c d e"; for str in $list
    • 用通配符读取目录: 文件名或路径名中使用通配符, shell 使用文件扩展匹配生成匹配指定通配符的文件名或路径名
      • 此时对循环变量的引用最好用双引号圈起来, 以免文件或路径名中包含空格产生错误
      • 可以在 for 命令中列出多个目录通配符, 将目录查找和列表合并进同一个 for 语句
    • 遍历命令输出: for 变量 in $(命令): for file in ${ls *.txt}
    • 使用 seq 命令生成数字序列: for i in $(seq 1 5)
    • 使用 {start..end..increment} 生成循环列表: for i in {1..5}
    • 无限循环 for ((;;)): 结合 break 与 continue

while

while 命令可以理解为 if-then 语句和 for 环境的混杂体. while 命令允许定义一个要测试的命令, 它会在每次迭代的一开始测试 test 命令. 如果测试命令返回的是退出状态码 0, 则循环执行一组命令; 如果测试命令返回非零退出状态码, 则停止执行那组命令

  • 基本格式:
  • 通常 test command 的退出状态码要随着循环中运行的命令而改变, 如检查循环命令中用到的 shell 变量的值. 如果退出状态码不发生改变, while 循环就将一起不停地进行下去
  • 使用多个测试命令, 只有最后一个测试命令的退出状态码会被用来决定什么时候结束循环
    • 每次迭代所有的测试命令都会被执行, 包括测试命令失败的最后一次迭代
    • 每个测试命令都出现在单独的一行上

安全与调试 (set)

set 用于设置或取消 Shell 的选项和变量

  • 设置 Shell 行为 set [选项]
  • 设置位置参数: set -- 参数1 参数2...
选项 含义
-e 一旦命令出错就退出脚本 (安全性高)
-n 语法检查但不执行命令 (测试脚本)
-u 使用未定义变量时报错 (避免拼写错误)
-x 调试模式, 执行前打印命令及参数 (常用于调试)
-o pipefail 管道中任一命令失败时整条命令失败
+选项 取消对应的选项
