Ubuntu 磁盘管理

最后更新于: 2026-01-07T09:23:06+08:00

通识

  • 建议对硬盘分区后使用, 便于管理数据并提升灵活性

基本命令

  • 查看分区情况: fdisk -l
    • 查看磁盘 sda 的分区: fdisk -l /dev/sda
  • 查看分区使用与挂载情况 (lsblk): df -h
  • 查看安装的磁盘类型与空间: lsblk -d -o NAME,SIZE,MODEL,TRAN,ROTA
  • 磁盘使用详情: ncdu or gdu

分区

fdisk (format disk) 磁盘分区管理的命令行工具, 适用于 MBR (MS-DOS) 和 GPT 分区表, 可以创建, 删除, 调整分区, 并修改分区类型: sudo fdisk [选项] <磁盘设备>

  • -l: 列出所有磁盘分区表 (不进入交互模式)
  • fdik <磁盘设备>: 进入交互模式
    • 常用: n 创建分区 → 根据情况输入 (大多数时候按回车选默认值) → p 打印分区, 确认无误 → w 保存设置
    • m 展示帮助
    • F 展示磁盘剩余未分区空间
    • n 增加一个新分区
    • d 删除分区
    • p 打印分区情况
    • t 修改分区类型 (Linux=83, swap=82, NTFS=7)
    • w 保存并退出
    • q 不保存退出
  • partprobe: 使修改生效

挂载

命令

  • umount [挂载点或设备路径]: 卸载已挂载的文件系统
    • -f: 强制卸载
    • -v: 显示详细操作信息
  • mount [选项] [设备] [挂载点]: 将存储设备挂载到文件系统的某个目录 (挂载点), 从而访问其中的数据
    • 设备: 磁盘分区 (/dev/sda1), UUID, LABEL 或网络存储
    • 挂载点: 一个已经存在的目录
    • findmnt/mount: 显示所有挂载点
    • -t <文件类型系统>: 指定文件系统类型
    • -o <挂载选项>: 指定挂载选项
    • -a: 挂载 /etc/fstab 中所有未挂载的设备 (测试 fstab 配置是否正确)
    • -v: 显示详细挂载信息
    • -l: 显示已挂载设备的 LABEL
    • -r: 以只读模式挂载
    • -w: 以读写模式挂载 (默认)
  • sudo blkid: 查看 UUID 和 LABEL (blok id)

/etc/fstab 文件

/etc/fstab (File System Table) 用于定义系统启动时自动挂载的文件系统和存储设备, 包含了系统需要挂载的所有分区, 网络共享和特殊文件系统的信息

文件由若干行组成, 每行定义一个挂载点, 字段之间使用单个制表符 (Tab) 分隔, # 表示注释, 每行必须包含 6 个字段: <设备标识> <挂载点> <文件系统类型> <挂载选项> <dump 备份标志> <fsck 检查顺序>

  • 设备标识: 设备路径 (/dev/sda1), UUID= (更可靠), LABEL=
  • 挂载点: 文件系统用于挂载的目录路径 (/, /home, /mnt/data)
  • 文件系统类型: ext4 (Linux 常用), xfs, btrfs, vfat (FAT32), ntfs (Windows NTFS), swap (交换分区), nfs (网络文件系统), tmpfs (临时文件系统)
  • 挂载选项: 多个选项用逗号分隔
    • defaults: (rw, suid, dev, exec, auto, nouser, async)
    • ro/rw: 只读/读写
    • noauto: 启动时不自动挂载
    • user: 允许普通用户挂载
    • nofail: 设备不存在时不报错
    • exec/noexec: 允许/禁止执行二进制文件
    • suid/nosuid: 允许/禁止 SUID 和 SGID 位
    • _netdev: 网络设备, 等待网络就绪后再挂载
  • dump 备份标志: 0 (不备份) (通常仅用于根分区), 1 (备份) (默认值, 适合大多数情况)
  • fsck 检查顺序:
    • 0 (不检查): 适用于物理磁盘, 虚拟文件系统, 远程目录
    • 1 (优先检查): 通常仅用于根分区, 启动分区
    • 2 (次要检查): 普通物理磁盘, 如 /home, /var
  • 验证配置正确性: sudo findmnt --verify
  • 重新加载: sudo mount -a

home 重挂载或扩容

Case 1: 新增一块设备合并使用

LVM 扩展

  • 命令:
    • 创建物理|逻辑卷: (pv|vg)create
    • 显示物理|逻辑卷: (pv|vg)display
    • 查看物理|逻辑卷空间: (vg|lv)s
  • 如果现有挂载设备已经是逻辑卷, 直接转到第 7 步, 将新增设备转化为物理卷后, 扩展已有卷组
  • 可以直接将磁盘转化为物理卷, 而不用先将磁盘分区
  1. 准备工作
  2. 将 /dev/sdb 初始化为物理卷: sudo pvcreate /dev/sdb
  3. 创建物理卷组: sudo vgcreate vg_home /dev/sdb
  4. 在卷组中创建逻辑卷: sudo lvcreate -l 100%FREE -n lv_home vg_home
  5. 格式化并挂载新逻辑卷:
  6. 同步数据: sudo rsync -aAXHS --partial --info=progress2 /home/ /mnt/new_home/
  7. 切换挂载点:
  8. 将原 /sda 加入卷组 (扩展卷组)
  9. 扩展逻辑卷:
  10. 永久挂载

Case 2: 完全使用一块新的独立设备

  1. 准备新硬盘

  2. 创建临时挂载点并挂载新分区

  3. 同步 /home 的数据 (保留所有权限)

  4. 重命名旧 /home 并创建新的挂载点 (留存原数据)

  5. 修改 /etc/fstab 实现自动挂载

  6. 挂载新 /home: sudo mount /home

  7. 验证成功后删除旧数据 (可选)

Case 3: 将当前被挂载设备扩容 (向后扩展)

前提: 当前被挂载设备是整块磁盘的其中一个分区或者仅被分配了部分空间

  • 当前挂载设备是起始分区, 直接转到第 4 步扩容
  • 新分区 (/dev/sda1) 是起始分区, 同步数据到新分区, 再扩容
  • 原则上可对起始分区直接扩容, 但建议扩容前备份数据
  1. 准备新分区 (/dev/sda1)
  2. 同步 /home 内容到新分区
  3. /home 重新挂载
  • 卸载
  • 挂载: 以防无法正常重启
  1. 删除原 /home 挂载分区 (/dev/sda2)
  2. 分区 (/dev/sda1) 扩容
