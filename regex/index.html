Shell 引号解析

理解:

  • 单引号: 完全不管, 原样输出
    • 固定正则表达式
  • 双引号: 稍微加工, 变量展示, 扩展 $, `, \ 转义字符 ($`"\), $() , 保护特殊字符 (空格*?单引号)
    • 含变量的正则表达式
  • 不加引号 (Shell 截胡): 过度加工, 替换变量, 拆分空格, 将特殊字符 (*?) 当成通配符匹配文件
    • [[ "string" =~ regex ]] 右侧的 regex 不能加引号, 否则正则符号 (.*) 会变成普通字符
    • 通配符, 重定向, 管道符

口决:

  • 有变量? 用双引号
  • 没变量? 用单引号
  • [[ =~ ]] 条件测试? 变量别加引号
  • 永远别完全不加引号

正则表达式

正则表达式 (Regular Expressions) 用于文本内容匹配 (如搜索, 替换字符串), 常见于 grep, sed, awk, vim 等命令

基础正则 (BRE)

  • 需转义特殊字符: {, }, +
  • .: 任意单个字符
  • *: 前一个字符 0 次或多次 (不包括换行)
  • .*: 匹配任意数量字符 (包括零个)
  • ?: 前一个字符 0 次或多次
  • +: 前一个字符 1 次或多次
  • ^: 行首
  • $: 行尾
  • []: 匹配任意一个 ([0-9], [a-zA-Z])
    • \d: 匹配数字 (\d+ 连续数字)
    • \w: 匹配单词字符 (字母, 数字, 下划线)
  • [^]: 匹配不在其中的字符
  • \: 转义特殊字符
  • {n,m}: 前一个字符 n 到 m 次
    • {n}: 恰好 n 次
    • {n,}: 至少 n 次
  • (pattern): 捕获分组
  • (a|b): 匹配 a 或 b

扩展正则 (ERE)

  • 无需转义, 易读

Perl 风格正则 (PCRE)

  • 无需转义
  • 支持命令:
    • grep -P
  • \d: 匹配数字
  • \D: 非数字
  • \s: 空白字符
  • \S: 非空白字符
  • \w: 单词字符 (字母, 数字, 下划线)
  • \W: 非单词字符 (标点符号, 空格)
  • \b: 匹配单词边界 (grep -P "\bcat\b" → cat)
  • 逻辑: | (或)

支持命令

  • grep: 文本搜索, 默认使用基础正则
    • grep -E 或 egrep: 启用扩展正则
  • sed: 流编辑器, 文本替换/处理, 默认使用基础正则
    • sed -E: 启用扩展正则
  • awk: 文本处理/字段提取, 默认使用扩展正则
  • find: 文件查找
    • -regex: 使用扩展正则
    • -name: 使用通配符
  • vim: 文本编辑器
    • \m: 基础正则
    • \v: 扩展正则
  • less: 分页查看, 默认使用基础正则

通配符

通配符 (Wildcards) 主要用于快速匹配文件名或目录名的特殊字符

注意

  • 区分大小写
  • * 不匹配以 . 开头的隐藏文件 (需显式指定 .*)
  • 通配符不需要加单引号, 加单引号表示禁用通配符, 即为普通字符
  • 特殊字符需通过 \ 转义: ^, ~

基础通配符

  • *: 任意数量 (包括零个) 的任意字符
    • *.txt: 所有 .txt 文件
  • ?: 单个任意字符
    • file?.log: file1.log, file2.log
  • []: 括号内任意一个字符或范围
    • [0-9], [a-z], [abc]
    • 扩展: [^/!] (取反)
  • {}: 生成组合 (非匹配, 用于展开)
    • {start..end}, {a,b,c} (不含空格)

zsh 扩展通配符

  • 检查当前与通配符相关的设置: setopt | grep glob (若输出 extendedglob 表示已启用)
  • 手动启用: setopt extendedglob (永久启用需写入 ~/.zshrc 中)
  • ^: 排除模式 (取反)
  • ~: 排除特定模式
    • *.txt~backup*: 匹配 .txt 文件但排除含 backup 的文件
  • (x|y): 匹配 x 或 y
